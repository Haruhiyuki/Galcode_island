  三、关键问题的直接答案

  同伴是否实现了后端 Agent 状态和回复的返回？

  状态返回：是。 run_stdout_loop() 逐行解析 stdout → reduce_event() 改变状态 → apply_side_effects() emit 事件。前端收到的 agent://status-changed 包含
  status（状态机状态）和 percent（进度）。

  回复返回：是，但不完整。 结果通过 agent://session-complete 返回，包含摘要/情绪/翻译结果/建议。但 OpenCode 路径因为缺失 spawn_opencode_process()
  无法编译，实际跑不起来。

  后端怎么启动 Agent？

  commands.rs::start_agent()
    → 用 LLM 把中文任务翻译成英文
    → 调用 manager::launch_opencode_agent() 或 launch_demo_agent()
    → config.rs 提供 AgentConfig（executable + args + env）
    → launcher.rs 用 std::process::Command 启动子进程，stdout=Stdio::piped()
    → 创建 AgentSession，记录 UUID
    → 开 std::thread 跑 run_stdout_loop()，逐行读 stdout

  前端如何与后端的返回对接？

  已对接：useAgentIPC.ts 监听了全部 7 个事件，写入 Zustand store。但 store 中的 summaryText/emotionText/resultZh/suggestion 只是存了，UI 没渲染。

  你需要做：在 MainView.tsx 中新增结果卡片，从 store 读取这四个字段展示。

  大模型的系统提示词应该放在哪里？

  现在的位置：src-tauri/src/llm/prompt.rs（Rust 硬编码的三个静态提示词）

  应该的位置：三层提示词，放不同地方：
  1. 翻译提示词（CN↔EN）→ 继续放 llm/prompt.rs，因为翻译逻辑是固定的
  2. 人物语气提示词（摘要+情绪生成）→ 应该从用户设置读取。你的 SettingsModal 已经有 "悄悄话 (System Prompt)" 字段，存入了 useSettingsStore（localStorage
  持久化）。需要通过 invoke("update_llm_settings", { systemPrompt }) 传给 Rust 后端，后端把它注入到 generate_summary_emotion() 的 system prompt 中
  3. 后续建议提示词 → 同上，需要可配置
