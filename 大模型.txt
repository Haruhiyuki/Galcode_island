 模式判断应该在哪里做？

  结论：后端大模型做模式判断，前端只负责按模式渲染。

  原因：模式判断需要理解 Agent 的完整输出内容，这件事 LLM 最擅长。而且模式判断可以和情绪生成用同一次 LLM 调用完成，不需要额外 API 开销。

  当前后端 generate_summary_emotion() 的调用已经是：
  用户中文输入 + Agent 英文结果(已译中) → LLM → { summary, emotion }

  只需让 LLM 多输出一个字段：
  用户中文输入 + Agent 英文结果(已译中) → LLM → { mode, summary, emotion }

  三个模式：

  ┌────────────┬──────────┬───────────────────────────────────────────┬─────────────────────────────────────┐
  │    模式    │   含义   │                 触发场景                  │              前端表现               │
  ├────────────┼──────────┼───────────────────────────────────────────┼─────────────────────────────────────┤
  │ normal     │ 正常完成 │ Agent 成功执行，结果可直接展示            │ 结果卡片 + 情绪文字 + GIF done      │
  ├────────────┼──────────┼───────────────────────────────────────────┼─────────────────────────────────────┤
  │ suggestion │ 提供建议 │ Agent 完成但 LLM 判断用户可能需要后续操作 │ 建议卡片（含选项按钮）+ GIF suggest │
  ├────────────┼──────────┼───────────────────────────────────────────┼─────────────────────────────────────┤
  │ error      │ 出错     │ Agent 报错、超时、无结果                  │ 错误卡片 + GIF error                │
  └────────────┴──────────┴───────────────────────────────────────────┴─────────────────────────────────────┘

  系统提示词应该放哪里？

  三层提示词，三个位置：

  第一层：翻译提示词（CN↔EN）
    → 放 llm/prompt.rs（Rust 硬编码）
    → 功能固定，不需要用户改

  第二层：人物设定提示词（决定语气、人设、称呼）
    → 用户在前端 SettingsModal 输入 → useSettingsStore 持久化
    → 通过 invoke("update_llm_settings") 发给 Rust
    → Rust 在调用 generate_summary_emotion() 时注入为 system prompt
    → 默认值放 llm/prompt.rs，用户可覆盖

  第三层：模式判断提示词（决定 normal/suggestion/error）
    → 附加在第二层提示词后面
    → 告诉 LLM 输出格式必须包含 mode 字段
    → 不建议用户修改，但可以随人设提示词一起发送

  实际 LLM 调用时的 system prompt 拼接：
  [用户自定义的人物设定]   ← 来自 SettingsModal
  +
  [模式判断指令]           ← 来自 llm/prompt.rs（追加，不暴露给用户）
  "请根据Agent执行结果判断模式(normal/suggestion/error)，
  并以{称呼}的口吻输出JSON: { mode, summary, emotion }"

  function ResultCard() {
    const mode = useAppStore(s => s.mode);
    const summaryText = useAppStore(s => s.summaryText);
    const emotionText = useAppStore(s => s.emotionText);
    const resultZh = useAppStore(s => s.resultZh);
    const suggestion = useAppStore(s => s.suggestion);

    if (!mode) return null;  // 还没收到结果时不渲染

    // 按模式选择不同容器样式
    const containerStyle = {
      normal:     "border-emerald-500/40 bg-emerald-50/60",
      suggestion: "border-amber-500/40 bg-amber-50/60",
      error:      "border-rose-500/40 bg-rose-50/60",
    }[mode];

    return (
      <motion.div className={`rounded-2xl border p-4 ${containerStyle}`}>
        {/* 情绪文字（春日口吻） */}
        <p className="text-lg">{emotionText}</p>

        {/* 正常模式 → 显示结果摘要 */}
        {mode === "normal" && resultZh && (
          <div className="mt-2 rounded-lg bg-white/60 p-3">
            <p className="text-sm whitespace-pre-wrap">{resultZh}</p>
          </div>
        )}

        {/* 建议模式 → 显示建议 + 选项按钮 */}
        {mode === "suggestion" && suggestion && (
          <div className="mt-2 space-y-2">
            <p className="text-sm">{suggestion}</p>
            <div className="flex gap-2">
              <button onClick={...}>按建议继续</button>
              <button onClick={...}>不用了</button>
            </div>
          </div>
        )}

        {/* 错误模式 → 显示错误信息 + 重试按钮 */}
        {mode === "error" && (
          <div className="mt-2 space-y-2">
            <p className="text-sm text-rose-600">{summaryText}</p>
            <button onClick={...}>重试</button>
          </div>
        )}
      </motion.div>
    );
  }

  完整数据流总结

  用户在 SettingsModal 设置:
    称呼 = "团长"
    系统提示词 = "你是凉宫春日，SOS团的团长..."

  用户在 InputBubble 输入任务:
    "帮我写个下载脚本"

  后端 start_agent:
    1. LLM 翻译 CN→EN: "Write a download script"
    2. 启动 OpenCode: opencode run --format json "Write a download script"
    3. 逐行解析 JSONL，emit agent://status-changed（进度）
    4. Agent 完成，拿到英文结果

  后端 LLM 后处理:
    5. translate_en_to_zh(英文结果) → 中文结果
    6. generate_summary_emotion(
         system = "你是凉宫春日..." + "输出mode+摘要+情绪",
         user   = 用户原始输入 + 中文结果
       )
       → LLM 返回 JSON: { mode: "normal", summary: "...", emotion: "搞定了团长~" }
    7. emit("agent://session-complete", { mode, summary, emotion, resultZh, suggestion })

  前端 useAgentIPC:
    8. 收到 → store.setMode("normal"), store.setEmotionText("搞定了团长~"), ...
    9. ResultCard 检测到 mode="normal" → 渲染绿色卡片 + 结果文本
    10. PetPreviewPanel 检测到 mode="normal" → 切换到 complete GIF

  你和同伴的边界很清晰：后端负责 mode 的判定（一次 LLM 调用同时产出 mode + summary + emotion），前端负责按 mode 渲染不同的 UI 卡片和 GIF。
