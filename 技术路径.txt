  三、核心数据状态 (State)

  3.1 全局状态 (Zustand Store)

  // store/useAppStore.ts — 全局应用状态
  interface AppState {
    // === 窗口管理 ===
    currentView: 'welcome' | 'main' | 'settings'
    windowConfig: {
      alwaysOnTop: boolean
      clickThrough: boolean       // 点击穿透 (休眠模式)
      opacity: number             // 0.0 ~ 1.0
    }

    // === Agent 管理 ===
    selectedAgent: 'claude-code' | 'opencode' | 'codex' | 'gemini' | 'cursor' | null
    agentStatus: AgentStatus      // idle | starting | running | thinking | waitingApproval | completed | error
    projectPath: string | null    // 用户选择的本地项目路径

    // === 任务状态 ===
    todos: TodoItem[]             // Agent 当前任务列表
    currentTool: string | null    // 当前正在执行的工具名 (Bash/Read/Edit/...)
    toolDescription: string | null // 当前工具的详细描述

    // === 日志 ===
    logEntries: LogEntry[]        // 最近 N 条日志

    // === 对话 ===
    currentSummary: string | null // 最近一次任务的 LLM 总结
    emotionText: string | null    // 情绪反馈文案

    // === Actions ===
    setAgent: (agent: string) => void
    setProjectPath: (path: string) => void
    updateAgentStatus: (status: AgentStatus, metadata?: AgentMetadata) => void
    updateTodos: (todos: TodoItem[]) => void
    appendLog: (entry: LogEntry) => void
    setSummary: (text: string, emotion: string) => void
  }

  3.2 Agent 状态机枚举 (参考 CodeIsland-main)

  // types/agent.ts
  type AgentStatus =
    | 'idle'              // 空闲等待
    | 'starting'          // 正在启动 Agent 进程
    | 'running'           // Agent 进程已启动，等待首条 hook 事件
    | 'thinking'          // Agent 正在思考 (LLM 推理中)
    | 'processing'        // 正在执行某个工具 (Bash/Read/Edit 等)
    | 'waitingApproval'   // 等待用户审批 (PermissionRequest)
    | 'completed'         // 当前任务完成
    | 'error'             // 发生错误

  // 人物 GIF 映射表 (状态 → GIF 文件路径)
  const STATUS_GIF_MAP: Record<AgentStatus, string> = {
    idle:              '/pet/idle.gif',

    running:           '/pet/ready.gif',
    thinking:          '/pet/thinking.gif',
    processing:        '/pet/working.gif',
    waitingApproval:   '/pet/question.gif',
    completed:         '/pet/happy.gif',
    error:             '/pet/sad.gif',
  }

  3.3 IPC 事件通道设计 (前端 ↔ Rust 双向)

  // 前端 → Rust (invoke)
  invoke('select_project_folder')               // → 打开系统文件夹选择器,返回路径
  invoke('launch_agent', { agent, cwd })        // → Rust 启动 Agent 子进程
  invoke('stop_agent', { sessionId })           // → 停止 Agent
  invoke('respond_permission', { decision })    // → 回复权限审批 (allow/deny)
  invoke('read_agent_logs', { sessionId })      // → 读取 Agent 的输出日志

  // Rust → 前端 (emit/listen)
  listen('agent://status-changed', (e) => {})   // ← 状态变化: idle/thinking/processing...
  listen('agent://tool-update', (e) => {})      // ← 工具执行开始: { tool, description }
  listen('agent://tool-result', (e) => {})      // ← 工具执行结果: { success, output }
  listen('agent://todo-update', (e) => {})      // ← 任务列表更新
  listen('agent://permission-request', (e) => {})) // ← 权限审批请求
  listen('agent://session-complete', (e) => {}) // ← 任务完成: { summary, emotion }
  listen('agent://log', (e) => {})              // ← 单条日志
  listen('agent://error', (e) => {})            // ← 错误信息

  3.4 关键数据类型

  interface TodoItem {
    id: string

    status: 'pending' | 'in_progress' | 'completed' | 'error'

  interface LogEntry {
    timestamp: number

    message: string
    toolName?: string
  }

  interface AgentMetadata {
    sessionId?: string
    toolName?: string
    toolDescription?: string
    cwd?: string
  }

  interface PermissionRequest {
    sessionId: string
    toolName: string
    toolDescription: string
    rawInput: Record<string, any>
  }

  前端工作：

  2. 将 src/ 内文件从 .jsx 升级为 .tsx
  3. 建立目录结构：

  4. 实现 App.tsx 的 window.label 路由分发 (参考 xsun_desktop_pet)

  交付物：透明无边框窗口 + 欢迎界面 + 主面板空壳。

  Phase 2 — 操作控制台 (预计 1-2 日)

  目标：实现顶部操作区，连通 Rust IPC 基础通道。

  前端组件：
  - ControlBar.tsx — 顶部操作栏
  - FolderPicker.tsx — 调用 @tauri-apps/plugin-dialog 打开文件夹选择
  - AgentSelector.tsx — 下拉菜单，支持 Claude Code / OpenCode / Codex / Gemini / Cursor
  - LaunchButton.tsx — 调用 invoke("launch_agent", ...)

  Rust 后端 (配合 Cursor 编写)：
  - 新增 select_project_folder 命令 → 调用 tauri-plugin-dialog
  - 新增 launch_agent 命令 → std::process::Command 启动 Agent CLI
  - 新增 stop_agent 命令 → 杀死子进程

  交互流程：
  [FolderPicker] → invoke('select_project_folder') → Rust dialog → 返回路径到前端
  [AgentSelector] → 更新 zustand store.selectedAgent
  [LaunchButton] → invoke('launch_agent', { agent: 'claude-code', cwd: '/path' })
                 → Rust spawn 子进程 → emit('agent://status-changed', 'starting')

  交付物：可打开文件夹、选择 Agent、点击启动 → Rust 侧能启动子进程并向前端 emit 状态。

  ---
  Phase 3 — 状态监控系统 (预计 2-3 日)

  目标：实时接收 Agent 状态、工具调用、Todo 进度，在 UI 上展示。

  核心 Hook: useAgentIPC.ts
  // 集中监听所有 agent:// 事件，分派到 zustand store
  function useAgentIPC() {
    useEffect(() => {
      const unlistens = [
        listen('agent://status-changed', handleStatusChange),
        listen('agent://tool-update', handleToolUpdate),
        listen('agent://todo-update', handleTodoUpdate),
        listen('agent://log', handleLog),
        listen('agent://session-complete', handleComplete),
      ]
      return () => { unlistens.forEach(fn => fn.then(u => u())) }
    }, [])
  }

  前端组件：
  - StatusMonitor.tsx — 监控区容器
  - AgentStatusBadge.tsx — 状态徽章 (带颜色+图标的状态指示器)
  - TodoProgress.tsx — 任务进度列表
  - LogStream.tsx — 最新日志滚动展示

  Rust 后端关键逻辑 (参考 CodeIsland-main 的 HookServer 模式)：
  - 监听 Agent 子进程的 stdout/stderr
  - 解析 Agent 输出的 hook 事件 (JSON 格式)
  - 事件路由：PreToolUse → agent://tool-update、PostToolUse → agent://tool-result、Notification → agent://log 等
  - 将 Agent 输出通过 app_handle.emit(...) 推送到前端

  交付物：启动 Agent 后，前端能实时看到状态变化、工具调用进度、日志流。

  ---
  Phase 4 — 人物视觉层 (预计 1-2 日)

  目标：实现人物 GIF 动图 + 状态机切换动画。

  前端组件：
  - PetCharacter.tsx — 人物容器 (固定在下半部分左侧)
  - CharacterGif.tsx — GIF 图片渲染 (<img> 标签，根据状态切换 src)
  - AnimationController.tsx — 状态机逻辑 (监听 agentStatus 变化)

  状态机映射 (扩展自 3.2 节)：
  idle              → idle.gif          (呼吸/待机动画)
  starting          → waking.gif        (苏醒/启动)
  running           → ready.gif         (就绪/等待)
  thinking          → thinking.gif      (思考中，如摸下巴、转圈)
  processing        → working.gif       (工作中，如敲键盘)
  waitingApproval   → question.gif      (疑问/等待中)
  completed         → happy.gif         (开心/庆祝)
  error             → sad.gif           (失落/错误)

  过渡动画 (framer-motion)：使用 AnimatePresence + motion.img 实现 GIF 切换时的淡入淡出效果。

  额外细节：
  - 窗口拖拽：在人物图片上绑定 onPointerDown/onPointerMove → window.startDragging() (参考 xsun_desktop_pet)
  - 双击交互：双击人物 → 显示/隐藏控制台

  交付物：人物 GIF 随 Agent 状态自动切换，带过渡动画，支持拖拽窗口。

  ---
  Phase 5 — 对话气泡层 (预计 1-2 日)

  目标：Agent 任务完成后展示 LLM 总结 + 情绪反馈。

  前端组件：
  - ChatBubble.tsx — 气泡外层 (定位在人物右侧)
  - BubbleContainer.tsx — 气泡容器 (带定位算法，避免遮挡人物)
  - BubbleContent.tsx — Markdown 内容渲染
  - EmotionFeedback.tsx — 情绪文字 (如 "任务完成！你很棒~ 🎉")

  气泡触发逻辑：
  agent://session-complete 事件到达
    → updateAgentStatus('completed')
    → setSummary(summaryText, emotionText)
    → 人物切换到 happy.gif
    → 气泡弹出 (framer-motion 从 scale(0) 弹入)
    → 10 秒后或用户点击其他地方，气泡自动收起

  气泡内容格式 (由 Rust/后端 LLM 处理)：
  ## 任务完成 ✅
  - 修改了 3 个文件
  - 新增了 `useAuth` hook
  - 修复了登录页面的样式问题

  > 你今天的效率很高，继续保持！

  交付物：任务完成后气泡自动弹出，展示总结内容，带弹性动画，自动消失。

  ---
  Phase 6 — 系统托盘与窗口管理 (预计 1 日)

  目标：系统托盘菜单、窗口显示/隐藏、设置面板。

  功能：
  - 系统托盘图标 (参考 BongoCat 的 useTray.ts)
  - 托盘菜单：显示主窗口 / 启动 Agent / 设置 / 退出
  - 窗口关闭行为：隐藏而非退出 (WindowEvent::CloseRequested → hide())
  - 设置面板 (SettingsView.tsx)：Agent 路径配置、外观主题

  交付物：完整的桌面集成体验 (托盘 + 窗口管理 + 设置)。

  ---
  Phase 7 — 多 Agent 并行与美化 (预计 2-3 日，可选)

  六、开发顺序建议

  Phase 1 (骨架+透明窗口)  →  Phase 2 (操作控制台+IPC)
                                      ↓
                                 Phase 3 (状态监控)  ← 核心难点：Agent 输出解析
                                      ↓
                Phase 4 (人物视觉层)  +  Phase 5 (对话气泡层)  可并行
                                      ↓
                                 Phase 6 (托盘+设置)
                                      ↓
                                 Phase 7 (美化增强, 可选)

  每个 Phase 的验证标准：
  - Phase 1: npm run tauri dev 看到一个透明无边框窗口
  - Phase 2: 点击"打开文件夹"能弹出系统对话框，点击"启动"能在终端看到 Rust 打印的进程信息
  - Phase 3: 启动 Agent 后，界面上实时显示状态徽章变化和日志
  - Phase 4: GIF 动图能随状态自动切换
  - Phase 5: 任务完成后气泡弹出，3 秒后自动消失
  - Phase 6: 关闭窗口→隐藏到托盘，右键托盘→菜单
